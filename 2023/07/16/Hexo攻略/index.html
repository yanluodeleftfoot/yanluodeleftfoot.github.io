前言

为什么使用Hexo搭建博客？搭建博客的目的

  • 因为我目前就只知道这个，且展示效果不错
  • 记录、整理学习的笔记，方便之后遗忘的时候翻看
  • 可以练习markdown语法的编写
  • 可以把博客部署在本地，之后通过http://localhost:4000/ 访问，很方便
  • 可以把博客部署到github上，之后也可以解锁很多玩法（还在探索中）
  • 等等

环境与工具准备

本教程主要面对的是Windows用户

  • 操作系统：Windows10、11
  • Node
  • Git
  • Hexo
  • 文本编辑器（推荐VS Code）
  • GitHub账号
  • 一个域名（需要购买，可选可不选）
  • 云服务器（可选）

安装Git

Git是目前世界上最先进的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。也就是用来管理你的Hexo博客文章，上传到Githhub的工具。

到Git官网上下载：https://gitforwindows.org/ (可能需要翻墙才能下载)

由于官网下载太慢可以通过淘宝的开源镜像下载 网址：https://registry.npmmirror.com/binary.html?path=git-for-windows/v2.41.0.windows.3/

下载安装完后鼠标右键有个Git Bash的命令行工具，以后就用这个工具来使用git。

以下内容是将博客部署到github上，如果你只是想做个学习笔记方便自己查看，那么以下步骤可以不做

GitHub创建个人仓库

首先，你得要有一个Github账户，去注册一个吧！

  1. 进入官网：https://github.com/

  1. 点击右上角的 Sign up(注册)

  1. 填写自己的邮箱、密码、用户名等信息，然后用邮箱验证即可完成。
  2. 注册完成后，点击右上角的+按钮，选择New repository，创建一个<用户名>.github.io的仓库。

  • 仓库的格式必须为：<用户名>.github.io (注意：前缀必须为用户名，不要等后面404了再来为什么！！！)
  • Description：为描述仓库（选填）
  • 勾选 Initialize this repository with a README 初始化一个 README.md 文件
  • 点击 Creat repository 进行创建

将hexo部署到GitHub

这一步，我们就可以将hexo和GitHub关联起来，也就是将hexo生成的文章部署到Github上，打开hexo博客文件夹，找到配置文件_config.yml，翻到最后，修改repository项为：

YourGitHubName 就是你的github账户

这个时候需要先安装deploy-git，也就是部署的命令，这样你才能使用命令部署到GitHub。

然后输入以下命令，将本地文件部署到GitHub上

hexo clean：清理你之前生成的东西，也可以不加

hexo g：全称为hexo generate，顾名思义，生成静态文件，可以缩写

hexo d：全程为hexo deploy，部署文章

注意：deploy时可能需要你输入username和password。

以上命令以后想要将本地文件同步至github上，都要运行一次

至此，博客的部署可以先告一段落了。至于之后的，比如你觉得域名不够炫酷，想要更换域名，觉得hexo自带的主题不好看，想要更好看的主题，诸如此类。那就自行百度搜索或者B站上搜索视频看吧！当然后面也有可能继续更新攻略，不过估计要一段时间
